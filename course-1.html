What the partner watches in the video (script)
Outline
5-minute walkthrough — one continuous take
  1. Open the CxE Guided Labeling Assistant. Pick Recommend sensitivity labels.
  2. Set industry and data context (e.g. healthcare / patient identifiers; or finance / customer PII).
  3. Walk the 6-question label decision flow and land on a defensible recommendation per content type.
  4. Review the recommended label set: Personal, Public, General, Confidential + AllEmployees, Highly Confidential.
  5. Confirm encryption defaults on the three sensitive labels (Confidential, Confidential\AllEmployees, Highly Confidential).
  6. Export the customer-ready label taxonomy. Module 2 will deploy these labels into the customer tenant via PowerShell.

What the partner watches in the video (script)
Outline
5-minute walkthrough — one continuous take
  1. Open the toolkit repo. Confirm PowerShell 7 (the script hard-fails on Windows PowerShell 5.1) and tenant-admin credentials for the target customer.
  2. Run .\Deploy-PurviewBestPractice.ps1 -TenantAdminUpn admin@customer.onmicrosoft.com -BPOnly -SkipDLP -WhatIf. Read the diff. Confirm only labels + tenant settings would change; DLP is skipped (handled in Course 2).
  3. Re-run without -WhatIf. Show the 5 sensitivity labels created in the Microsoft Purview portal: Personal, Public, General (default), Confidential + AllEmployees sub-label, Highly Confidential.
  4. Show tenant settings enabled: Unified Audit Log on, SharePoint AIP integration on, PDF labelling on, label co-authoring on.
  5. Flag the sharp edge: encryption identity AuthenticatedUsers includes B2B guests and OTP users. Partners should audit the guest list before publishing the encrypting labels.
  6. Re-run is safe (idempotent) — every object is stamped with [Managed by SMBTool Purview Toolkit] so re-runs detect and skip what already exists.
How to read this mockup. Each module is a single activity (Module 1 = design, Module 2 = deploy).
